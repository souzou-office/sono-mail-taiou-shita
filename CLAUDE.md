# 開発進捗

## 概要
Gmailの未返信メールを自動検出し、AI（Claude）で優先度判定・要約を行うWebアプリ。
GAS（Google Apps Script）をバックエンドに使用し、フロントはReact（Vite）でCloudflare Pagesにデプロイ。

## 実装済み機能

### コア機能
- GASで未返信メールを定期スキャン（毎朝7時 / 直近24時間）
- Claude Sonnetによるメール仕分け（返信要否判定・優先度・感情検知・要約）
- 返信待ち追跡（自分が送って相手から返事がないメール）
- 返信済み自動検知（repliedフラグ）
- 3時間おきの返信チェック（checkReplies）

### UI
- Airtable風テーブルレイアウト（未対応 / 返信待ちタブ）
- 優先度・ステータス順ソート（至急 > 要注意 > 未対応 > 低）
- メール本文プレビュー展開
- Gmailへの直接リンク（Open）
- スマホ対応レスポンシブレイアウト（768px以下でカード型）

### 操作・管理
- 済み・不要ボタン（Undo付き、5秒以内に取り消し可能）
- 除外データのGAS同期（PC間で共有）
- 学習機能（不要にした送信者を3回以上で自動除外）
- 監視メールアドレスの設定（フロントから変更可能）
- APIトークン認証

### インフラ
- Cloudflare Pagesデプロイ
- GitHub Pagesへの自動デプロイ（CI）
- ファビコン

## バグ修正履歴（主要なもの）
- MY_EMAIL空文字時の誤判定防止
- 至急(urgent)が偽値扱いされるソートバグ
- 不要・済みボタンの状態がリロードで戻る問題 → localStorage → GAS同期
- category:primaryがGoogle Workspaceで無効 → 削除
- 設定保存のCORS問題 → POSTからGETに変更
- scanEmailsで新規0件時に既存データが全消しされるバグ（#34で修正）
